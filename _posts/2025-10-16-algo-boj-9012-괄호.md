## Introduction
백준 온라인 저지(BOJ)의 **[괄호](https://www.acmicpc.net/problem/9012)** 문제 풀이입니다.

## Problem Description
> [문제 링크](https://www.acmicpc.net/problem/9012)

<p>괄호 문자열(Parenthesis String, PS)은 두 개의 괄호 기호인 ‘(’ 와 ‘)’ 만으로 구성되어 있는 문자열이다. 그 중에서 괄호의 모양이 바르게 구성된 문자열을 올바른 괄호 문자열(Valid PS, VPS)이라고 부른다. 한 쌍의 괄호 기호로 된 “( )” 문자열은 기본 VPS 이라고 부른다. 만일 x 가 VPS 라면 이것을 하나의 괄호에 넣은 새로운 문자열 “(x)”도 VPS 가 된다. 그리고 두 VPS x 와 y를 접합(concatenation)시킨 새로운 문자열 xy도 VPS 가 된다. 예를 들어 “(())()”와 “((()))” 는 VPS 이지만 “(()(”, “(())()))” , 그리고 “(()” 는 모두 VPS 가 아닌 문자열이다. </p>

<p>여러분은 입력으로 주어진 괄호 문자열이 VPS 인지 아닌지를 판단해서 그 결과를 YES 와 NO 로 나타내어야 한다. </p>

## Approach
### 1. 문제 분석 (Problem Analysis)
이 문제는 주어진 괄호 문자열(PS)이 올바른 괄호 문자열(VPS)인지 판단하는 문제입니다.
- VPS의 조건:
    1.  여는 괄호 `(`와 닫는 괄호 `)`의 개수가 같아야 합니다.
    2.  닫는 괄호 `)`가 나올 때, 그에 대응하는 여는 괄호 `(`가 이전에 존재해야 합니다.
- 가장 최근에 들어온 여는 괄호와 짝을 맞춰야 하므로 **스택(Stack)** 자료구조(LIFO)가 가장 적합합니다.

### 2. 해결 아이디어 (Solution Idea)
1.  문자열을 왼쪽부터 한 글자씩 순회합니다.
2.  **여는 괄호 `(`**: 스택에 넣습니다(Push).
3.  **닫는 괄호 `)`**:
    - 스택이 비어있지 않다면, 스택에서 하나를 꺼냅니다(Pop). (짝을 맞춤)
    - 스택이 비어있다면, 짝이 맞는 여는 괄호가 없는 것이므로 **NO**입니다.
4.  모든 문자열을 순회한 후:
    - 스택이 비어있으면 모든 괄호가 짝을 맞춘 것이므로 **YES**입니다.
    - 스택에 남은 괄호가 있다면 짝이 맞지 않은 것이므로 **NO**입니다.

### 3. 코드 분석 (Code Analysis)
- `stack` 리스트를 사용하여 스택을 구현합니다.
- `valid` 플래그를 사용하여 중간에 유효하지 않음이 판별되면 반복을 중단합니다.
- 최종적으로 `valid`가 `True`이고 `stack`이 비어있어야만 `YES`를 출력합니다.

## Solution
```python
import sys

# 입력 받기
input = sys.stdin.readline
T = int(input())

for _ in range(T):
    ps = input().strip()
    stack = []
    valid = True
    
    for char in ps:
        if char == '(':
            stack.append(char)
        else:  # char == ')'
            if stack:
                stack.pop()
            else:
                # 스택이 비어있는데 닫는 괄호가 나온 경우
                valid = False
                break
    
    # 1. 중간에 오류가 없었고 (valid == True)
    # 2. 스택에 남은 괄호가 없어야 함 (not stack)
    if valid and not stack:
        print("YES")
    else:
        print("NO")
```

## Complexity Analysis
- **Time Complexity**: $O(N)$
    - 문자열의 길이 $N$만큼 한 번 순회합니다. 각 문자에 대한 스택 연산은 $O(1)$입니다.
- **Space Complexity**: $O(N)$
    - 최악의 경우(모두 여는 괄호인 경우), 스택에 문자열 길이만큼 저장됩니다.

## Conclusion
스택(Stack)의 가장 대표적인 응용 문제입니다. 괄호 쌍 맞추기 문제는 컴파일러의 구문 분석이나 수식 계산 등 다양한 분야에서 활용되는 중요한 개념입니다.
