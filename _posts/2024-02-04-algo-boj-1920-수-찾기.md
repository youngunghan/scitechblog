## Introduction
백준 온라인 저지(BOJ)의 **[수 찾기](https://www.acmicpc.net/problem/1920)** 문제 풀이입니다.

## Problem Description
> [문제 링크](https://www.acmicpc.net/problem/1920)

<p>N개의 정수 A[1], A[2], …, A[N]이 주어져 있을 때, 이 안에 X라는 정수가 존재하는지 알아내는 프로그램을 작성하시오.</p>

## Approach
### 1. 문제 분석 (Problem Analysis)
이 문제는 $N$개의 정수 리스트 $A$ 안에 특정 정수 $X$가 존재하는지 확인하는 문제입니다.
- $N$의 범위는 최대 100,000입니다.
- $M$개의 수에 대해 존재 여부를 확인해야 하며, $M$ 역시 최대 100,000입니다.
- 단순한 선형 탐색(`in` 연산자 사용 시 리스트 탐색)은 $O(N)$이 소요되므로, 전체 시간 복잡도는 $O(NM)$이 되어 시간 초과(TLE)가 발생합니다.
- 따라서 $O(\log N)$의 시간 복잡도를 가지는 **이분 탐색(Binary Search)** 또는 $O(1)$의 **해시 집합(Set)**을 사용해야 합니다.

### 2. 해결 아이디어 (Solution Idea)
두 가지 효율적인 방법이 있습니다.

1.  **이분 탐색 (Binary Search)**:
    - 리스트 $A$를 먼저 정렬합니다 ($O(N \log N)$).
    - 각 쿼리에 대해 이분 탐색을 수행합니다 ($O(M \log N)$).
    - Python의 `bisect` 라이브러리를 사용하면 편리합니다.

2.  **해시 집합 (Set)**:
    - 리스트 $A$를 `set`으로 변환합니다 ($O(N)$).
    - 각 쿼리에 대해 존재 여부를 확인합니다 ($O(M \times 1)$).
    - 구현이 더 간단하고 빠를 수 있습니다.

### 3. 코드 분석 (Code Analysis)
이전 코드에서 주석 처리된 부분(`for m in ms: ...`)은 리스트에서의 선형 탐색을 의미하며, 이는 시간 초과의 원인이 됩니다.
아래의 정답 코드는 **이분 탐색**을 활용하여 효율적으로 값을 찾아냅니다.

## Solution

```python
import sys
from bisect import bisect_left

input = sys.stdin.readline

# 입력 받기
N = int(input())
ns = list(map(int, input().split()))
ns.sort() # 이분 탐색을 위한 정렬

M = int(input())
ms = list(map(int, input().split()))

# [시간 초과 발생하는 코드]
# for m in ms:
#     print(1 if m in ns else 0)

# 각 쿼리에 대해 이분 탐색 수행
for m in ms:
    idx = bisect_left(ns, m)
    # bisect_left는 삽입할 위치를 반환하므로, 
    # 1. 인덱스가 범위 내에 있고 (idx < N)
    # 2. 해당 위치의 값이 찾는 값과 같아야 함 (ns[idx] == m)
    if idx < N and ns[idx] == m:
        print(1)
    else:
        print(0)
```

## Complexity Analysis
- **Time Complexity**: $O((N+M) \log N)$
    - 정렬: $O(N \log N)$
    - $M$번의 이분 탐색: $O(M \log N)$
- **Space Complexity**: $O(N)$
    - 입력 데이터를 저장하는 리스트 공간
